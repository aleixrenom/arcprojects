# Project Instructions

Use this document as the authoritative implementation context for code changes in this repository. When asked to modify code, follow the architecture and current status below and do not invent extra features.

For the current task, implement only the requested change within the existing architecture; do not add items from the 'Next' list unless the user explicitly requests them in the same message.

## Overview

Portfolio website for Aleix Renom Cisa with two content streams: apps and projects. The current UI has one Apps page and one Projects page, each showing a card grid, plus a full-screen detail view opened from a card. You move between pages by clicking the vertical separator on the left (for the Projects page) or right (for the Apps page) edge of the screen.

See `Design/DESIGN.md` for the full design spec.

## Tech Stack

- React + TypeScript + Vite
- Zustand for global state
- CSS custom properties for theming
- If the user asks to implement URL-linked pages or routing, add only the minimal router support required for that request; otherwise keep the current non-router behavior. Do not add TanStack Router in the current implementation task.
- The project may use the Motion library for animation work in a future task; do not add animation dependencies in the current task unless the user explicitly requests them.

## Architecture

- `src/App.tsx` — root, renders either page grid or `DetailPage`
- `src/store/ui.ts` — `theme`, `activePage`, `selectedCard`, `openCard()`, `closeCard()`
- `src/pages/AppsPage.tsx` / `ProjectsPage.tsx` — card grids
- `src/pages/DetailPage.tsx` — blank full-screen detail page
- `src/index.css` — global theme and interaction styles
